## 2b. Troubleshooting the launch itself

If the QR scans but the app never reaches the sign-in screen, match the symptom on-device to the recovery below **before** re-scanning. Re-scanning the same QR against a broken Metro won't help.

### Symptom A — Blank white screen with a lone blue spinner (Expo Go bundle-load spinner never finishes)

Expo Go connected to Metro but the bundle never arrived. In decreasing order of likelihood:

1. **Metro isn't actually running.** Confirm the terminal window shows `Metro waiting on exp://…` — if it exited or was closed, restart: `npx expo start --go --clear --port 8082`.
2. **Firewall blocked port 8082 for `node.exe`.** First Windows Firewall prompt was dismissed. Allow Private-network access, or run this once in an elevated shell:
   ```powershell
   New-NetFirewallRule -DisplayName "Expo Metro 8082" -Direction Inbound -Protocol TCP -LocalPort 8082 -Action Allow
   ```
3. **LAN IP rotated.** Re-run `qa\generate-expo-qr.ps1` to refresh the QR against the current adapter IP.
4. **Phone on a different Wi-Fi network than the laptop** (or the laptop's Wi-Fi is on a guest SSID with client isolation). Verify both are on the SAME SSID.

Fallback if none of the above stick:
```powershell
npx expo start --go --tunnel --clear --port 8082
```
Tunnel mode routes through ngrok and works even across networks — slower but bulletproof for a stuck session.

### Symptom B — Blue "Something went wrong. Sorry about that." screen with "View error log"

Metro delivered a bundle and the JS threw during first evaluation. This is NOT a QR problem.

1. Tap **View error log** on the crash screen — the stack trace is the answer. Screenshot the first frame and paste it into the follow-up.
2. Most common root cause on SDK 54: a stale Metro bundle. Fix in the CLI:
   ```powershell
   # Stop the current expo start (Ctrl+C), then:
   npx expo start --go --clear --port 8082
   ```
   In Expo Go: pull down the notification shade → **Reload JS bundle**, OR force-quit Expo Go and re-scan.
3. Second most common: **Expo Go SDK version mismatch**. Open Expo Go → Profile → About — confirm SDK 54 is listed as supported. If not, update Expo Go from the Play Store; the Play-Store build supports the current + N-1 SDKs, older sideloaded APKs will crash on first-JS against a newer project.
4. If steps 1-3 don't clear it, run the compile gauntlet on the laptop to confirm the shipped bundle actually parses:
   ```powershell
   cd fan-wave-app
   npx tsc --noEmit
   npx expo-doctor
   ```
   Any TS or expo-doctor error there is the crash source. Fix before re-launching.

Only proceed to §2 once the app reaches the sign-in screen cleanly.
