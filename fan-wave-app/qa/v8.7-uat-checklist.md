### 0d. (Optional) Confirm ESPN cron is alive
Run in SQL Editor — if this returns 0 rows with `status='succeeded'` in the last hour, the WC games will not repopulate after the wipe in migration 063:

```sql
SELECT j.jobname, jrd.status, jrd.start_time, jrd.return_message
  FROM cron.job j
  JOIN cron.job_run_details jrd ON jrd.jobid = j.jobid
 WHERE j.jobname LIKE '%sync%'
 ORDER BY jrd.start_time DESC
 LIMIT 10;
```

If cron is broken, follow the runbook in migration 060's header (CRON_SHARED_SECRET ↔ vault.fan_wave_service_role_key drift).

### Test #3 — Soccer Cup tab no fake teams (root: seeded fixtures from migration 006 re-emerging)

### Test #4 — Bulls Nation Chicago not in Home → Your Groups (root: global cache key bleeding membership across logins)
**What changed**: `hooks/useData.ts` `useMyGroups` now scopes its AsyncStorage cache to `user.id`; migration 063 cleans server-side memberships for reviewer accounts.

### Test #5 — Watch party appears on Discover tab immediately (root: Discover used useState + loadData() not React Query; invalidation was firing but on a key Discover never subscribes to)
**What changed**: `app/(tabs)/discover.tsx` now uses `useFocusEffect` to refetch on tab focus AND `subscribeToWatchParties` for live INSERT notifications.
